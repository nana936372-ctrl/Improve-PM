# AI 产品经理思维训练器设计

## 背景与目标

本产品是一个面向“想转 AI 产品经理”或“希望提升 AI 产品思维”的 Web 训练工具。首版目标是做成完整单机 MVP：用户可以注册登录，配置 OpenAI-compatible 模型服务，完成真实训练题，获得 AI 评分、面试官追问、参考答案，并持续沉淀个人能力成长记录。

产品核心不是做题库堆量，而是让 AI 产品经理思维变得可训练、可反馈、可追踪。首版以响应式网页为主，桌面端提供完整长文本训练体验，移动端支持同流程单列使用。

## 首版范围

首版必须包含：

- Supabase Auth 登录、注册、退出和会话保护。
- AI 产品经理通用能力训练，覆盖需求洞察、AI 能力边界、指标设计、Prompt/RAG/Agent 产品化、风险治理等主题。
- 两类题型：选择题快练和案例深训。
- OpenAI-compatible AI 配置，支持 `baseURL`、`apiKey`、`model`，并支持 `.env.local` 作为兜底配置。
- 题目生成、用户答案评估、AI 面试官追问、参考答案生成。
- 训练记录、总分趋势、能力维度趋势、短板与下一步建议。
- 本地开发和 Vercel 部署两种运行方式。

首版不包含：

- 付费、会员等级、组织空间或多人协作。
- 运营后台、公开题库社区、分享广场。
- 复杂异步队列、通知系统、流式输出。
- 原生移动 App。
- 第三方 AI key 的云端多设备同步加密方案。

## 用户与核心场景

目标用户包括三类：

- 想转 AI 产品经理的求职者，需要系统训练案例分析和面试追问。
- 已经做产品但缺少 AI 产品化经验的人，需要补齐 AI 能力边界、指标和风险治理。
- 想搭建 AI 产品作品集或面试准备材料的人，需要可复盘的训练记录。

核心场景是“单题深训”：用户选择能力维度和难度，系统生成一个真实业务场景题。用户完成作答后，AI 按 Rubric 输出评分和反馈，再根据短板追问 1 到 3 轮。训练结束后，系统生成参考答案和复盘建议，并把本次表现写入成长记录。

选择题用于快速训练概念判断、方案取舍、指标优先级、风险识别和 AI 能力边界判断。选择题也写入能力趋势，但不进入长轮追问。

## 产品结构

首版页面包括四个区域：

- 认证页：登录、注册、忘记密码入口预留。未登录用户只能访问认证页。
- 训练台：核心页面，完成选择题快练和案例深训。
- 成长记录：查看历史训练、趋势图、维度短板和下一次训练建议。
- 设置页：配置 AI 服务、模型、默认训练偏好和追问轮数。

训练台在桌面端采用三栏布局：

- 左栏：训练配置、题型、能力维度、难度、训练进度。
- 中栏：题目正文、选项或长文本答题区、提交按钮、草稿状态。
- 右栏：AI 评分、选项解析、Rubric 反馈、追问、参考答案摘要。

移动端采用同信息结构的单列布局。配置区可折叠，评分与追问按训练阶段向下展开。所有长文本、按钮和卡片必须在桌面和移动端不溢出、不遮挡。

## 训练流程

选择题快练流程：

1. 用户选择能力维度、难度和题型。
2. 服务端调用 AI 生成单选或多选题，返回题干、选项、正确或较优答案、选项解析和能力标签。
3. 用户选择答案并提交。
4. 系统计算选择题得分，并结合 AI 解析输出本题反馈。
5. 系统保存训练会话、用户答案、评分结果和能力快照。
6. 成长记录更新总分和维度趋势。

案例深训流程：

1. 用户选择能力维度、难度和场景偏好。
2. 服务端调用 AI 生成真实业务案例题，包含背景、目标、约束、交付要求和 Rubric。
3. 用户输入主观答案，系统自动保存本地草稿。
4. 用户提交答案后，服务端调用 AI 按 Rubric 评分，返回总分、维度分、证据化点评、优势、短板和追问建议。
5. 用户回答 1 到 3 轮 AI 面试官追问。
6. 服务端基于完整上下文生成参考答案、常见误区、复盘总结和下一次训练建议。
7. 系统保存完整训练会话、追问记录、评分和能力快照。

## Rubric 与能力维度

首版固定 6 个 Rubric 维度：

- 用户与业务洞察：是否理解用户、场景、业务目标和真实约束。
- 问题定义与目标拆解：是否能把模糊需求拆成清晰问题、目标和优先级。
- AI 能力边界判断：是否理解模型能力、幻觉、数据依赖、RAG/Agent/Prompt 的适用边界。
- 方案设计与取舍：是否能提出可落地方案，并说明关键取舍。
- 指标与实验设计：是否能定义北极星指标、过程指标、评估方法和实验路径。
- 风险、伦理与治理：是否识别安全、合规、隐私、偏见和体验风险，并提出治理方式。

案例题每个维度按 0 到 20 分评分，总分 100 分。选择题按题目映射的能力维度写入部分分值，支持单选和多选。多选题需要处理漏选、错选和全错，不只做二元对错。

AI 评分必须输出结构化 JSON。每个维度至少包含 `score`、`maxScore`、`evidence` 和 `advice`，避免只有空泛鼓励。

## Prompt 模板

首版包含四类 Prompt 模板：

- 题目生成模板：输入题型、能力维度、难度、场景偏好，输出题干、选项或案例背景、Rubric、参考考察点。
- 答案评估模板：输入题目、用户答案、Rubric，输出总分、维度分、证据点评、优势、短板、建议。
- 追问模板：输入题目、答案和评分短板，输出一个聚焦追问、追问意图和期望补充点。
- 参考答案模板：输入完整训练上下文，输出优秀答案框架、常见误区和下一次训练建议。

所有 AI 接口优先要求 JSON 输出。服务端使用 schema 校验。若 AI 返回非法 JSON，服务端尝试一次修复；仍失败时返回明确错误，并保留用户答案和草稿。

## 技术架构

技术栈：

- Next.js App Router。
- TypeScript。
- Supabase Auth 与 Supabase Postgres。
- `@supabase/ssr` 和 `@supabase/supabase-js`。
- OpenAI-compatible Chat Completions API。
- Tailwind CSS 或项目创建时选定的轻量 UI 样式体系。
- 图表使用轻量图表库或自定义 SVG/Canvas，首版只需要趋势线和维度雷达/条形图。

运行形态：

- 本地开发：Next.js 本地运行，Supabase 云端项目提供 Auth 和数据库。
- Vercel 部署：通过环境变量配置 Supabase 与默认 AI Provider，回调 URL 按部署域名配置。

AI 调用只发生在服务端 Route Handlers 或 Server Actions 中。浏览器不直接请求第三方 AI Provider。Supabase `service_role` 不进入客户端。客户端只使用 Supabase publishable/anon key。

## Supabase 设计

表结构：

- `profiles`：用户资料，与 Supabase `auth.users` 一对一。
- `ai_settings`：用户级 AI 偏好，保存 baseURL、model、默认追问轮数等非敏感配置。
- `training_sessions`：一次训练会话，记录题型、状态、难度、能力标签、总分、开始和完成时间。
- `session_questions`：题目内容，记录题型、题干、选项、案例背景、Rubric、参考答案。
- `user_responses`：用户答案，记录选择题选项或主观题正文，支持草稿和提交状态。
- `evaluations`：评分结果，记录总分、维度分、选项解析、证据点评、优势、短板和建议。
- `followup_turns`：案例深训追问轮次，记录问题、用户回答、AI 追问意图和补充评价。
- `ability_snapshots`：训练完成后的能力维度快照，用于成长趋势。

安全策略：

- 所有公开 schema 下的用户数据表开启 RLS。
- 用户只能 `select/insert/update/delete` 自己 `user_id = auth.uid()` 的记录。
- `profiles` 创建可通过触发器或首次登录时 upsert 完成。
- 不使用用户可编辑的 metadata 做授权判断。
- 不把 Supabase `service_role` 或 AI Provider 密钥暴露到浏览器。

AI API key 存储：

- 首版优先支持页面输入后保存在浏览器本地，并由用户发起训练时传给服务端请求使用。
- `.env.local` 可配置默认 `AI_BASE_URL`、`AI_API_KEY`、`AI_MODEL`，作为开发者和部署兜底。
- `ai_settings` 不保存明文第三方 API key。若未来需要多设备同步 API key，需要单独设计加密和密钥管理方案。

## 错误处理

必须处理的错误状态：

- 未登录或会话过期：跳转登录页，保留本地草稿。
- AI 配置缺失：引导用户进入设置页，说明缺失项。
- AI 请求超时或限流：展示可重试错误，不清空答案。
- AI 返回结构不合法：服务端尝试修复一次，失败后展示错误并保留上下文。
- Supabase 写入失败：提示保存失败，允许用户重试保存。
- RLS 权限失败：展示通用权限错误，日志中记录诊断信息。

用户作答草稿应优先保存在浏览器本地，提交成功后再写入 Supabase。任何 AI 或网络错误都不能导致用户答案丢失。

## 测试策略

单元测试：

- Prompt 输入构造。
- AI JSON schema 校验和修复逻辑。
- 选择题评分，包含单选、多选、漏选、错选。
- Rubric 评分结果到能力快照的映射。
- 训练状态流转。

组件测试：

- 登录态下训练台渲染。
- 选择题作答与解析展示。
- 案例题答题、评分、追问和参考答案展示。
- 设置页配置优先级：页面配置优先，环境变量兜底。
- 成长记录趋势展示空状态和有数据状态。

端到端验证：

- 注册、登录、退出。
- 完成一轮选择题快练。
- 完成一轮案例深训。
- AI 配置缺失时的引导。
- 桌面和移动端关键页面截图检查。

## 交付标准

首版完成后，用户应能在本地启动项目，配置 Supabase 与 OpenAI-compatible Provider 后完成完整训练闭环。部署到 Vercel 后，同一套功能可通过公网网页访问。

完成标准：

- 登录注册可用。
- 训练台可生成选择题和案例题。
- 用户答案可提交并获得结构化反馈。
- 案例题可完成至少一轮 AI 追问。
- 参考答案和复盘建议可生成。
- 训练记录按用户隔离保存。
- 成长记录可展示总分和 6 个维度趋势。
- 关键错误状态有明确 UI 反馈。
- README 包含本地运行、Supabase 配置和 Vercel 部署说明。
